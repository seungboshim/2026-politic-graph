# 정치성향 테스트 웹사이트 — 설계 문서

**날짜**: 2026-06-08
**상태**: 사용자 승인 대기

## 1. 개요

질문을 따라가며 답하면 세분화된 한국 정치성향 유형과 가장 가까운 실명 정치인을 알려주는 웹 테스트.

- **톤**: 하이브리드 — 입구는 가볍고 재밌게, 결과는 의외로 정교하게. 질문 18~22개.
- **핵심 가치**: 단일 좌우축이 아닌 다축 태그 시스템으로, 찬탄/반탄·친명/비명·부정선거론 같은 한국 특화 균열을 잡아낸다.
- **차별점**: 유형 분류와 정치인 매칭이 같은 태그 벡터 공간에서 독립 계산됨.

## 2. 태그 스키마

사용자와 정치인이 **동일한 스키마의 벡터**를 가진다.

### 2.1 연속 축 6개 (−100 ~ +100)

| 축 | − 방향 | + 방향 | 비고 |
|---|---|---|---|
| 경제 | 분배·복지·규제 | 시장·성장·감세 | 고전적 좌우 |
| 사회문화 | 다양성·차별금지법·개방 | 전통·가족·질서 | |
| 대북·외교 | 화해협력·자주외교 | 대북강경·한미일 동맹 | |
| 제도신뢰 | 불신 (선거·사법·언론) | 신뢰 | 극좌·극우 모두 낮게 나옴. 부정선거론·검찰불신을 같은 축의 양쪽 버전으로 포착 |
| 젠더 | 페미니즘 동조 | 반페미니즘 | 2030 핵심 균열. 좌우와 독립 |
| 정치관여 | 무관심 | 고관여 | 무당층/무관심층 구분용 보조축. 정치인 매칭에서는 제외 |

### 2.2 조건부 스탠스 태그 (분기 진입자만 채움)

| 태그 | 값 | 누가 답하나 |
|---|---|---|
| 탄핵 스탠스 | 찬탄 / 반탄온건 / 윤어게인 (범주형) | 우향 분기 |
| 부정선거론 | 배척 0 ~ 수용 100 | 반탄 분기 |
| 이재명 평가 | 강성친명 0 ~ 비명 100 | 좌향 분기 |
| 검찰·사법개혁 | 온건 0 ~ 강경 100 | 좌향 분기 |

**설계 결정:**
- 정치인은 모든 태그를 가짐. 사용자는 지나간 분기의 태그만 가짐 → 매칭 거리는 사용자가 답한 차원만으로 계산.
- 탄핵 스탠스가 범주형인 이유: 정도 문제가 아닌 진영 정체성. 연속값으로 뭉개면 무의미한 중간값이 나옴.
- 좌향 사용자에게 탄핵 질문을 하지 않는 이유: 변별력 없음(거의 전원 찬탄). 좌측 변별축은 친명/비명·검찰개혁 강도.
- 이민·기본소득 같은 개별 이슈는 별도 축이 아니라, 해당 질문이 기존 축에 점수를 주는 방식으로 흡수.

## 4. 질문 플로우 (적응형 분기 + 태그 점수)

총 18~22문항, 3단계.

### Phase 1 — 공통 측정 (8~10문항, 전원 동일)
- 6개 연속 축을 거칠게 측정. 문항 하나가 1~2개 축에 점수를 줌.
- 형식 혼합: 리커트 5단계 + 상황 선택형(밈스러운 선택지, 내부적으로 정확한 태그 점수).
- 라우터 질문 1개 포함: "그나마 마음 가는 정당은?" — 분기 결정에 큰 가중치.

### Phase 2 — 진영 분기 (6~8문항)
Phase 1 누적 점수 + 라우터 질문으로 3갈래:
- **좌향**: 이재명 평가(친명/비명) → 검찰개혁 강도, 노동·정의당계 감지, 페미니즘 심화
- **중도·무당**: 무당층 vs 무관심 구분, 이준석계 신호(반페미+시장) 감지
- **우향**: 탄핵 스탠스(핵심 관문) → 반탄이면 부정선거론·윤어게인 심화 (분기 속 분기), 찬탄이면 개혁보수 vs 전통보수

### Phase 3 — 확정 심화 (3~4문항)
- 후보 유형 2~3개로 좁혀진 상태에서, 그 유형들을 가장 잘 가르는 질문 투입.

### 엣지 케이스
- **무관심층 조기 감지**: Phase 1에서 정치관여 점수 바닥이면 짧은 전용 트랙으로.
- **교차 성향**(경제 좌 + 사회 우 등): 분기는 종합점수로 하나만 타지만 축 점수는 보존 → 최종 매칭에서 반영.
- **뒤로가기**: 답 수정 허용. 수정 지점 이후 점수 재계산.

## 5. 매칭 알고리즘

```
d(user, target) = √( Σ wᵢ · (userᵢ − targetᵢ)² )   // 축별 0~1 정규화, 답한 차원만
```

- **범주형(탄핵) 거리 행렬**: 찬탄↔반탄온건 0.6, 찬탄↔윤어게인 1.0, 반탄온건↔윤어게인 0.4
- **가중치**: 탄핵 2.0, 부정선거 1.5, 이재명 평가 1.5 > 연속 축 1.0
- **유형 판정** = 13개 유형 중 최근접. **정치인 매칭** = 동일 공간 최근접 Top 3 + 유사도 %.
- 유형과 정치인은 독립 계산 → 교차 결과 허용 ("찬탄 개혁보수인데 가장 가까운 정치인은 이준석").
- 유사도 % = `(1 − d/d_max) × 100`, 상위 매칭이 70~95% 구간에 오도록 보정. 100%는 나오지 않게.
- 정치관여 축은 정치인 매칭에서 제외.

## 6. 정치인 데이터 (~30명)

```ts
{
  id: "...",
  name: "...",
  party: "...",
  vector: { ... },        // 유형과 동일 스키마, 모든 태그 보유
  evidence: [             // 선택 필드 — 있으면 좋고 없어도 동작
    { tag: "탄핵", value: "찬탄", source: "2024-12 담화", url: "..." },
  ]
}
```

- 태깅 근거(발언·표결·인터뷰 출처)는 **선택 필드**. 있는 것만 결과 페이지에 노출 — 실명 매칭의 방어선.
- 좌파 인물 풀이 구조적으로 빈약(정의당 몰락 이후)하므로 인지도 있는 인물(권영국 등)로 보강.
- 2026-06-03 지방선거 직후 유동기 — 인물 거취 변동에 대비해 데이터 파일 수정 + 재배포로 대응.

## 7. 결과 페이지 · 댓글 · 통계

### 결과 페이지 (`/r/[결과ID]`)
제출 시 익명 결과 저장 + 고유 URL 발급. 구성(위에서부터):
1. 유형 카드 — 유형명 + tagline + 설명 + 키워드 해시태그
2. 정치인 매칭 Top 3 + 유사도 % + 인물별 근거 펼쳐보기 (근거 있는 태그만)
3. 6축 레이더 차트 — 교차 성향이 시각적으로 드러남
4. 통계 한 줄 — "전체 응답자 중 8.2%가 당신과 같은 유형"
5. 댓글창
- 무당층·무관심 유형은 "그나마 가까운 정치인" 톤으로 조절.
- 유형별 정적 OG 이미지 13장 — 카톡·X 공유 시 유형명 노출. (동적 개인화 카드는 추후)

### 댓글
- **테스트 완료자만 작성** — localStorage의 결과 ID 필요. 읽기는 전체 공개.
- **뱃지 자동 부착**: `[강성 친명 개혁파 · 이재명 92%] 닉네임: 내용`
- 익명 닉네임 + 비밀번호(해시 저장), 본인 비번으로 삭제. 회원가입 없음.
- 공용 광장 하나 + 유형별 필터. (유형별 분리는 초기에 텅 비어 보여서 제외)
- 모더레이션: 신고 버튼, 금칙어 필터, IP 기반 속도 제한, 관리자 삭제는 Supabase 대시보드에서 직접.

### 통계 (`/stats`)
- 유형 분포 차트, 정치인 매칭 1위 분포. 집계 쿼리 + 60초 캐시.
- 저장 데이터에 개인 식별 정보 없음. IP는 스팸 방지용 해시만.

## 8. 아키텍처

### 스택
- **Next.js (App Router) + TypeScript + Tailwind**, Vercel 배포
- **Supabase (Postgres) + Drizzle ORM** — 댓글 모더레이션을 Supabase 대시보드로 해결(관리자 UI 불필요), 추후 Realtime 확장 여지. ORM은 Drizzle로 유지해 이식성 확보.
- 차트: 경량 SVG 레이더 직접 구현 또는 recharts

### 구조 — 콘텐츠와 코드의 분리

```
/data
  questions.ts    ← 질문·선택지·태그 점수·분기 규칙
  types.ts        ← 13개 유형
  politicians.ts  ← 정치인 ~30명
/lib
  scoring.ts      ← 채점 엔진 (순수 함수)
  branching.ts    ← 분기 결정 (순수 함수)
/app
  page.tsx        ← 랜딩
  test/           ← 질문 플로우 (클라이언트 실시간 분기)
  r/[id]/         ← 결과 + 댓글
  stats/          ← 통계
  (server actions)← 결과 저장, 댓글 CRUD, 신고, 통계 집계
```

- 정치 지형 변화 → `/data`만 수정 후 재배포. 관리자 UI는 MVP 제외.
- 진행 중 채점은 클라이언트(실시간 분기 필요), 최종 제출 시 서버가 같은 엔진으로 재계산 후 저장(조작 방지).

### DB 테이블

```
results   (id, type_id, vector, top_politicians, created_at)
comments  (id, result_id, nickname, password_hash, body, created_at, deleted, report_count)
```
통계는 results 집계 쿼리 + 캐시. 별도 테이블 없음.

## 9. 에러 처리

- **DB 장애 시 우아한 강등**: 채점은 클라이언트에서 완결 → 결과는 무조건 표시, 저장·댓글·통계만 비활성 안내.
- 존재하지 않는 결과 ID → 404 + "테스트 하러 가기".
- 시크릿 모드(localStorage 불가) → 댓글 작성 불가 안내.
- 새로고침 시 진행 상태 복원(sessionStorage).
- 중복 제출 방지.

## 10. 테스트 전략

- **페르소나 골든 테스트** (핵심): 13개 유형마다 모범 답안 페르소나를 정의하고, 그렇게 답하면 반드시 해당 유형 + 예상 정치인이 상위로 나오는지 단위 테스트. 질문·가중치 수정 시 회귀 방지.
  ```ts
  test("강성 친명 페르소나 → 강성 친명 개혁파 + 이재명 1위")
  test("반탄+부정선거 페르소나 → 광장 우파 + 전한길/황교안 상위")
  ```
- 분기 로직 단위 테스트 (경계값 포함).
- E2E 스모크 1개: 테스트 완주 → 결과 표시 → 댓글 작성 (Playwright).

## 11. MVP 범위

**포함**: 테스트 본체(18~22문항, 적응형 분기), 13유형 + 정치인 ~30명 매칭, 결과 페이지(레이더 차트·통계 한 줄·OG 이미지), 결과 저장 + 통계 페이지, 뱃지 댓글 시스템.

**제외 (추후)**: 동적 개인화 공유 카드, 관리자 UI, 유형별 댓글방, Realtime 댓글, 정치인 상세 페이지, 광장 우파 세분화.
