# [Gemini 프롬프트 0/3] 공통 컨텍스트 — 먼저 이 메시지를 보내세요

당신은 한국 정치 도메인 전문가이자 심리측정(psychometric) 설문 설계 컨설턴트입니다. 저는 "정치성향 테스트" 웹앱의 데이터를 정교화하려 합니다. 이 메시지는 **배경 모델**이고, 이어지는 3개 메시지에서 (1) 유형, (2) 정치인, (3) 질문 데이터를 차례로 검토·개선해 달라고 요청할 겁니다. 지금은 **읽고 이해만** 하고, "이해했다 + 모델에서 의심되는 점"만 짧게 답해주세요.

## 무엇을 만드나
사용자가 18문항 내외를 풀면 → 6개 연속 축 + 4개 조건부 스탠스로 된 **벡터**가 나오고 → 13개 정치 유형 중 최근접 유형과, 별도로 가장 가까운 실명 정치인 Top3가 매칭됩니다.

## 6개 연속 축 (모두 −100 ~ +100)
- `econ` 경제: − 분배·복지·규제 / + 시장·성장·감세
- `social` 사회문화: − 다양성·차별금지·개방 / + 전통·질서·가족
- `security` 대북외교: − 화해·자주외교 / + 대북강경·한미일동맹
- `trust` 제도신뢰: − 불신(선거·사법·언론) / + 신뢰  ※ 극좌·극우 모두 낮게 나오는 축. 부정선거론(우)과 검찰·언론 불신(좌)을 같은 축의 양극단으로 잡음
- `gender` 젠더: − 페미니즘 동조 / + 반페미니즘  ※ 좌우와 독립적인 2030 균열
- `engage` 정치관여: − 무관심 / + 고관여  ※ 무당층 vs 무관심 구분용. **정치인 매칭에선 제외**(정치인은 전원 고관여)

## 4개 조건부 스탠스
- `impeach` (범주형): `'pro'`(찬탄) · `'antiMild'`(반탄온건) · `'yoonAgain'`(윤어게인). 진영 정체성이라 연속값 아님.
- `fraud` (0~100): 부정선거론 0 배척 ~ 100 수용
- `leejm` (0~100): 이재명 평가 0 강성친명 ~ 100 비명/반이재명
- `prosec` (0~100): 검찰·사법개혁 0 온건 ~ 100 강경(해체)
※ 사용자 답변에서 스탠스는 기준값 50에서 ± 누적되지만, **유형·정치인의 타깃 벡터에 적을 땐 절대값(0~100)** 으로 적습니다. 유형/정치인은 자기 진영에서 안 묻는 스탠스는 생략 가능(유형), 정치인은 4개 모두 채웁니다.

## 매칭 알고리즘 (이미 구현됨 — 바꾸지 마세요. 데이터가 이 규칙 위에서 작동함)
- 거리 = 가중 정규화 유클리드. **사용자가 답한 차원 ∩ 타깃이 가진 차원만** 사용.
- 축은 차이를 200으로, 스탠스 숫자는 100으로 나눠 정규화.
- `impeach` 범주 거리행렬: pro↔antiMild 0.6, pro↔yoonAgain 1.0, antiMild↔yoonAgain 0.4.
- 기본 가중치: 축 1.0, `impeach` 2.0, `fraud`/`leejm`/`prosec` 1.5. 유형마다 `weights`로 특정 축·스탠스 가중치를 덮어쓸 수 있음(판별력 강화).
- 유사도% = (1−정규화거리)×100, 상한 97. 유형 판정은 engage 포함 + 유형 weights 적용, 정치인 매칭은 engage 제외.

## 분기/라우팅 (질문 설계 시 중요)
- 라우트 점수 = (econ+social+security)/3 + q9(정당 선호) routeBonus.
- `engage ≤ −50`(Phase1 기준)이면 **무관심 트랙**. 그 외 점수 ≤ −18 좌향 / ≥ +18 우향 / 사이 중도.
- Phase1(전원 공통 9문항) → Phase2(진영별 6~8문항) → Phase3(후보 유형 가르는 3문항). 우향은 r1(탄핵)이 관문이고 반탄이면 부정선거 분기로 더 들어감.

## 확정된 결정(바꾸지 마세요)
- 정당은 **9종만** 사용: 더불어민주당·국민의힘·개혁신당·조국혁신당·진보당·정의당·새로운미래·자유통일당·무소속. (정의당계 군소=기본소득·사회민주·녹색은 "정의당"으로 통합)
- 디자인/엔진/스키마는 완료. 이 작업은 **콘텐츠 값(벡터·문구·태그)** 만 다룸.
